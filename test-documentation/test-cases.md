### Tested Website
[SweetShop Demo](https://sweetshop.netlify.app/sweets)

I performed manual testing on SweetShop, covering main website functionality with structured test coverage.
Testing included functional, exploratory, and UI/UX checks, for both valid and invalid user scenarios.

**Testing Approach:**
- Functional Testing: verified all website features work as intended
- Positive Scenarios: happy path flows such as adding products to the cart, correct city selection, header navigation links (expected to work)
- Negative Scenarios: invalid user actions such as login with unregistered credentials, invalid email input, incorrect cart calculations
- Exploratory Testing: checked for unexpected behaviors, UI inconsistencies, and edge cases
- UI/UX Checks: confirmed elements are displayed correctly and interactive components behave as expected

**Summary of findings:**
- Test Cases include both Positive and Negative scenarios, showing structured approach to QA
- Failures from Negative tests are documented in Bug Reports in Notion
- Positive tests with PASS confirm that core functionality works correctly
- Overall, the approach demonstrates realistic manual testing workflow, covering functional correctness, usability, and error handling

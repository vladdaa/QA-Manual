# SweetShop Manual Testing

This repository contains QA documentation for **SweetShop Demo**, including **manual testing checklists, test cases, and bug reports**.  

The testing approach covers both **positive and negative scenarios**, exploratory testing, and UI/UX verification to ensure proper functionality and realistic quality control workflow.

## Tested Website

### Overview
- **Positive Scenarios:** functionality works as expected (adding products to cart, header navigation, city selection by country)  
- **Negative Scenarios:** invalid inputs or incorrect actions (login with unregistered user, email validation, cart calculation errors)  
- **Environment:** Windows 10, Chrome v.146.0.7680.153, Desktop  

This documentation demonstrates **structured manual testing workflow** suitable for portfolio presentation, showing coverage of main website functionality, validation, and error handling.
